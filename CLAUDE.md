# รับหิ้วคุ้มไหม — เว็บเทียบราคาสินค้าทั่วโลก

## ภาพรวมโปรเจกต์
เว็บเปรียบเทียบราคาสินค้า (น้ำหอม/เครื่องสำอาง-สกินแคร์/กระเป๋า/นาฬิกา) ระหว่างไทย ญี่ปุ่น เกาหลีใต้ ฝรั่งเศส สหรัฐฯ โดยดึงราคาจากเว็บ official ของแต่ละแบรนด์ในแต่ละประเทศ แปลงเป็นเงินบาท ไว้ช่วยตัดสินใจว่าจะ "รับหิ้ว" จากประเทศไหนคุ้มสุด เป็นไฟล์ static HTML ล้วนๆ ไม่มี backend

## กฎเหล็กที่ต้องยึดตลอด
**ราคาทุกตัวต้องยืนยันได้จริงจากเว็บ official ของแบรนด์ในประเทศนั้นๆ เท่านั้น ห้ามเดา/ประมาณ** ถ้าเช็กราคา/ไซส์ไม่ได้ ให้ระบุว่า "ยังไม่ยืนยัน" หรือไม่ใส่แถวนั้นเลย ดีกว่าใส่ตัวเลขที่ไม่แน่ใจ

## ไฟล์ในโปรเจกต์ (4 ไฟล์ ต้อง sync กันเสมอ)
- `price-compare.html` — ไฟล์ต้นฉบับภาษาไทย (master)
- `price-compare-en.html` — ไฟล์ต้นฉบับภาษาอังกฤษ (master)
- `index.html` = `price-compare.html` + ลิงก์ไปหน้าอังกฤษ (สำหรับ deploy จริง เป็นหน้าแรกของเว็บ)
- `en.html` = `price-compare-en.html` + ลิงก์กลับหน้าไทย (สำหรับ deploy จริง)

**ทุกครั้งที่แก้เนื้อหา/ฟีเจอร์ ต้องแก้ที่ 2 ไฟล์ master ก่อน (TH+EN คู่กันเสมอ) แล้วค่อย regenerate index.html/en.html ใหม่จาก master** วิธี regenerate (ต้องใช้วิธี "overwrite content" ไม่ใช่ rename ไฟล์ เพราะบางระบบไฟล์บล็อกการ rename ทับไฟล์เดิม):
```bash
perl -0pe 's{(</header>)}{<a href="en.html" class="lang-switch">English version →</a>\n$1}' price-compare.html > /tmp/index_new.html
cat /tmp/index_new.html > index.html
perl -0pe 's{(</header>)}{<a href="index.html" class="lang-switch">ภาษาไทย →</a>\n$1}' price-compare-en.html > /tmp/en_new.html
cat /tmp/en_new.html > en.html
```
แล้ว diff เช็คว่าต่างกันแค่บรรทัดลิงก์ภาษาเท่านั้น (ลิงก์ใช้ class `.lang-switch` ที่นิยามไว้ใน `<style>` ของแต่ละ master — ไม่ใช่ inline style อีกต่อไปหลังรีดีไซน์ UI)

## โครงสร้างข้อมูลสินค้า (ในแท็ก `<script>` ของแต่ละไฟล์)
- `PRODUCTS` array: แต่ละ product มี `id, brand, category, name/nameTh, sizeOrder[], defaultSize, sizes{ "<size>": [...rows] }, sizeNote?`
- หมวดหมู่ (`category`) ที่ใช้อยู่: น้ำหอม/Fragrance, เครื่องสำอาง-สกินแคร์/Makeup & Skincare, กระเป๋า/Bags, นาฬิกา/Watches
- แถวราคาแต่ละประเทศสร้างด้วย `R(country,flag,currency,localPrice,thb,retailer,url,isBase,refundPct)` — `isBase=true` สำหรับแถวไทย (ใช้เป็นราคาฐานเทียบ %), `refundPct` = อัตราคืนภาษีนักท่องเที่ยวโดยประมาณ (0.10 ญี่ปุ่น, 0.065 เกาหลีใต้, 0.115 ฝรั่งเศส) ใช้แสดงราคาประมาณหลังคืนภาษีเท่านั้น ไม่ใช้ตัดสิน "ถูกที่สุด"
- อัตราแลกเปลี่ยนคงที่ที่ใช้ทั้งโปรเจกต์ (Wise mid-market, เช็ก ก.ค. 2026): 1 JPY=0.2017, 1 KRW=0.02199, 1 EUR=37.82, 1 USD=33.33 THB

## ฟีเจอร์หลักที่ทำไปแล้ว
1. เทียบราคา 11 สินค้า × 5 ประเทศ พร้อม badge "ถูกที่สุด" (`getCheapestNonBase()` — อิงราคายืนยันจริงเท่านั้น)
2. ตัวคำนวณราคาโดยประมาณหลังคืนภาษีนักท่องเที่ยว (เลือกประเทศ → กรอกราคา → โชว์ผลลัพธ์เฉพาะประเทศนั้น) รวมเพดานซื้อสูงสุดต่อวัน/ต่อครั้งของญี่ปุ่น (¥500,000/ร้าน/วัน สำหรับของใช้สิ้นเปลือง) และเกาหลีใต้ (₩1,000,000/ใบเสร็จ, ₩600,000/ร้าน/วัน, ₩5,000,000/ทริป) — ฝรั่งเศสไม่พบเพดานที่ยืนยันได้ ระบุตรงๆ ว่าไม่มีข้อมูลแทนการเดา
3. Nav หมวดหมู่/แบรนด์ (รีดีไซน์ ก.ค. 2026 รอบล่าสุด): อยู่ใน sidebar ฝั่งซ้าย (`#sidebarNav`, sticky บนจอ ≥900px, ย้ายมาอยู่บนสุดแบบ stack บนมือถือ) เป็นปุ่ม pill ทรงแคปซูล ไม่ใช่ accordion บาร์เต็มความกว้างแบบเดิม — `renderSidebar()` สร้างปุ่มหมวดหมู่ (`.cat-pill`) เรียงตามแนวตั้ง กดแล้ว toggle เปิด/ปิด (เก็บสถานะด้วย `expandedCategories` Set) หมวดที่เปิดจะโชว์ปุ่มแบรนด์ (`.brand-pill`, เก็บสถานะด้วย `expandedBrands` Set คีย์ `"หมวด|แบรนด์"`) แทรกอยู่ใต้ปุ่มหมวดนั้นเลย ส่วน `renderGrid()` แยกหน้าที่ไปเรนเดอร์แค่การ์ดสินค้า (มี label เล็กๆ บอกหมวด·แบรนด์กำกับแต่ละกลุ่ม) — เปิดได้พร้อมกันหลายหมวด/หลายแบรนด์เหมือนเดิม (multi-select ไม่ใช่ single-select filter) sidebar อ่านจาก PRODUCTS เต็มเสมอ ไม่ขึ้นกับคำค้นหา ส่วนตอนพิมพ์ค้นหา `renderGrid()` จะสลับเป็น flat list แบนราบแทน (sidebar ยังโชว์ปกติ)
4. เคยมีฟีเจอร์ "ช่องทางอื่นที่ถูกที่สุด" (duty-free) แต่ผู้ใช้ให้ตัดออกแล้วทั้งหมด (RA() function ถูกลบไปแล้ว) — ไม่ต้องเพิ่มกลับมาเว้นแต่ถูกขอใหม่
5. รูปสินค้า: field `image` ใน product object เป็นได้ 2 แบบ — path รูปจริง (เช่น `images/chanel-chance-eau-tendre-edp.jpg`, เก็บไฟล์ไว้ในโฟลเดอร์ `images/`) หรือ emoji เดี่ยวๆ (fallback ตอนหารูปจริงไม่ได้) ฟังก์ชัน `renderThumb(image, cls)` เช็คว่ามี `/` ในสตริงไหมเพื่อตัดสินใจว่าจะ render เป็น `<img>` หรือ emoji div — สินค้า Dior/YSL/La Mer 5 ตัวยังเป็น emoji อยู่เพราะเว็บแบรนด์พวกนี้มีระบบกันบอทบล็อกการดึงรูปอัตโนมัติ (ลองผ่าน Claude-in-Chrome แล้วก็ยัง permission denied) ไว้ลองใหม่วันหลัง
6. ดีไซน์ปัจจุบัน (รีดีไซน์ ก.ค. 2026): แนว Apple/Notion/Airbnb — สีพื้นหลังอุ่นๆ `#faf8f5`, การ์ดขาวมุมโค้ง เงานุ่ม, accent เขียว `#0b7a5b` ใช้เน้นราคาถูกสุด/segmented control ของไซส์, ปุ่ม "ไปที่ร้าน" เป็นลิงก์ตัวหนังสือสีเขียวธรรมดา (ไม่ใช่ปุ่มทึบ ตามฟีดแบ็กที่บอกว่าเด่นเกินไป)
7. ราคาเปรียบเทียบ (รีดีไซน์ ก.ค. 2026 รอบล่าสุด): เลิกใช้ `<table>` แล้ว เปลี่ยนเป็นการ์ดแนวนอน `.pricecard` ต่อประเทศ (คลาส `.pricecards` ครอบ) แบ่ง 3 โซน — ซ้าย (ธง+ชื่อประเทศ+badge), กลาง (ราคาท้องถิ่นตัวเล็ก + ราคาบาทตัวหนา), ขวา (badge ส่วนต่าง% + แหล่งข้อมูล+ลิงก์ร้าน + วันที่เช็ก + หมายเหตุคืนภาษีตัวจิ๋ว) การ์ดที่ถูกสุดมีพื้นเขียวอ่อน (`.pricecard-best`) บนมือถือ (`max-width:640px`) จะยุบเป็น 1 คอลัมน์ stack แนวตั้งอัตโนมัติ

## สถานะ deploy ปัจจุบัน
- Netlify site เชื่อมกับ GitHub repo แล้ว (auto-deploy เวลา repo อัปเดต) — แต่ต้องอัปโหลดไฟล์ 4 ไฟล์ล่าสุดเข้า repo เอง (ยังไม่มี automation เขียนเข้า GitHub ให้อัตโนมัติจากฝั่ง Claude/Cowork)
- **เป้าหมายที่ผู้ใช้ต้องการตอนนี้**: ให้ Claude Code (ซึ่งรันบนเครื่องผู้ใช้จริง มี git + credentials ของผู้ใช้อยู่แล้ว) ช่วย commit + push ไฟล์เข้า repo ให้อัตโนมัติ แทนที่จะต้องอัปโหลดผ่านหน้าเว็บ GitHub เองทุกครั้ง — นี่คือสิ่งที่ Cowork ทำให้ไม่ได้ (ไม่มี GitHub credentials/tool) แต่ Claude Code ทำได้ถ้า repo นี้ clone อยู่ในเครื่องแล้วและตั้ง git remote/credential ไว้แล้ว

## Known nuance ที่ควรรู้ก่อนแก้ต่อ
- ไทยเป็น "ราคาฐาน" (isBase) เสมอ ไม่ปรับ refund
- สหรัฐฯ ราคาที่แสดงไม่รวม sales tax อยู่แล้ว (บวกตอน checkout ตามรัฐ) ไม่ต้องปรับเพิ่ม
- Dior.com ใช้หน้าเดียวกันทุกไซส์ (เลือกด้วย JS) — ยืนยันราคาต่อไซส์แม่นยำได้ยากกว่า chanel.com บางสินค้าจึงมีแค่ไซส์เดียวที่ยืนยันได้
- chanel.com เป็นโดเมนกลาง จะ redirect ตาม IP ผู้เข้าดูเสมอ (ลิงก์คลิกอาจเด้งแม้ราคาที่ใช้จะยืนยันถูกต้องแล้ว)
